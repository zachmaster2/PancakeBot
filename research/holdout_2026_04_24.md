## Promotion rule (binding going forward)

A future candidate strategy variant must satisfy ALL of:

1. **5-fold CV baseline clearance.** Beat the current CV baseline on the
   standard 6-criterion framework (5/5 positive, PnL margin, PnL/DD floor,
   interaction consistency, bracket check, holdout sign-test).
2. **Holdout clearance.** Run the candidate on epochs 474880–475311 with
   the candidate's config. Report bets, WR, PnL. The candidate must not
   regress vs the current baseline on the holdout, OR (if the candidate
   claims to fix a pathology that the baseline exhibits on the holdout)
   must explicitly name the pathology and show the baseline's failure
   mode.

Failing #2 is a promotion blocker even if #1 passes cleanly. The whole
point of the frozen holdout is to catch strategies that overfit to the
CV structure.

## Extending the holdout

Every time we run `--sync` on a new day, we add more rounds. Those new
rounds become part of an EXTENDED holdout automatically — train+val
stays frozen at 474879, holdout grows. Future doc updates should append
a row to the "Holdout (frozen)" boundary with the new upper bound and
the sync date, so the holdout history is auditable.

When the holdout grows past a useful size (rough heuristic: >= 10k
rounds or 3x the smallest CV fold), consider promoting the oldest slice
of holdout INTO train+val as a new fold and starting a fresh holdout.
That's a deliberate decision that requires a written rationale, not an
automatic rollover.
