The Game

PancakeSwap Prediction

A binary prediction game running on the BNB Smart Chain. Every 5 minutes, a new round opens. Players bet whether the BNB price will go up (Bull) or down (Bear) before the round locks. If you're right, you split the losing side's pool (minus a 3% fee). PancakeBot automates this by reading real-time market data and placing bets when it detects momentum.

System Architecture
Signal Pipeline -- How It Decides
Primary

BTC Multi-Timeframe Agreement

Detects short-term BTC momentum using three lookback windows on 1-second candles from OKX.

  • BTC 3s, 7s, and 15s returns must all agree in direction
  • Minimum strength: min(|r3|, |r7|, |r15|) ≥ threshold
  • Pool-adaptive threshold: 0.0002 for small pools, 0.0001 for large
  • ETH/SOL confirmation boosts sizing when they agree with BTC direction
Regime-2

ETH + SOL Agreement

When BTC is silent (no multi-TF agreement), checks whether ETH and SOL independently agree on direction.

  • Both ETH and SOL multi-TF(3,7,15) must fire in the same direction
  • Minimum strength: min(eth_str, sol_str) ≥ 0.00015
  • Smaller sizing than primary: base_frac=0.02, cap=0.5 BNB
  • Fires on ~37% more rounds, filling flat BTC periods

Position Sizing

Continuous adaptive sizing based on signal strength, cross-pair confirmation, and payout odds (Kelly reasoning).

Runtime Loop -- One Round
Epoch Alignment
Query BSC for current epoch and round timing. Identifies the open round (currently accepting bets) and the just-locked round (awaiting settlement).
NTP-sync Wake
Force a fresh NTP query against a public stratum-2 pool (cloudflare/google/pool.ntp.org, rotated round-to-round). Apply the (local − ntp) offset for the rest of the round so critical-path scheduling uses freshly-corrected clock. 5s wake budget vs. ~125ms p99 query — deliberately generous for robustness against environmental drift.
lock_at − ntp_sync_wakeup_offset_ms → ~11.10s before lock
Bankroll Wake
Refresh wallet balance: live mode = BSC RPC call (~50-200ms p99); dry mode = in-memory simulated bankroll. Feeds the risk gates and decide_open_round with fresh-truth right before the bet decision. 5s wake budget covers any plausible RPC stall.
lock_at − bankroll_wakeup_offset_before_lock_ms → ~6.10s before lock
Critical-path Wake
Single critical-path entry. Sequentially: read pool aggregate from WSS subscriber (BetBull/BetBear events with block_ts < lock_at − pool_cutoff_seconds, in-memory ~5ms) → fire 3 parallel OKX /history-candles GETs (BTC/ETH/SOL; BNB disabled, see MomentumGate._OKX_SYMBOLS_FETCHED) → compute multi-TF signal + sizing. The bet-submit deadline gate at lock_at − bet_submit_deadline_offset_before_lock_ms gates the actual TX submission below.
lock_at − critical_path_wakeup_offset_before_lock_ms → ~1.10s before lock
Timing Guard + Execute
Abort if decision-ready is within bet_submit_deadline_offset_before_lock_ms of lock_at (TX submitted that close would land after lock and revert). Otherwise submit bet (receipt-waited with bet_tx_receipt_timeout_seconds ≈ 35s). Live mode: clamp to min bet when clamp_bet_to_contract_minimum=true.
lock_at − bet_submit_deadline_offset_before_lock_ms → ~750ms before lock
Claim Wake
Sleep until previous round's close + buffer (35s post-close). Epoch handshake (both modes). Dry: settle simulated bets. Live: batched claim scan via getUserRounds; each claim TX is receipt-waited with claim_tx_receipt_timeout_seconds ≈ 35s. Revert (status=0) and timeout outcomes log a WARN entry and POST a PancakeBot-live CLAIM FAILED alert to the live Discord webhook; no retry — next iteration's cursor walk re-detects any still-claimable epochs.
close_at(claim_epoch) + 35s
All wake offsets derive from empirical constants in pancakebot/timing_constants.py; not user-tunable. User-tunable knobs in config.toml [runtime]: kline_cutoff_seconds, pool_cutoff_seconds, max_consecutive_kline_fetch_failures.
Operating Modes
--live

Live Mode

Real-time trading with real BNB. Requires BSC_WALLET_PRIVATE_KEY. clamp_bet_to_contract_minimum clamps sizes for safe testing.

--dry

Dry Mode

Real-time paper trading. No private key needed. --fresh / --no-archive flags for state management.

Backtest Mode

Offline replay with equity curve generation.

--sync

Sync Mode

Data sync from Graph + OKX + chain constants.
