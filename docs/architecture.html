The Game

PancakeSwap Prediction

A binary prediction game running on the BNB Smart Chain. Every 5 minutes, a new round opens. Players bet whether the BNB price will go up (Bull) or down (Bear) before the round locks. If you're right, you split the losing side's pool (minus a 3% fee). PancakeBot automates this by reading real-time market data and placing bets when it detects momentum.

System Architecture
Signal Pipeline — How It Decides
Tier 1 — Primary

BNB Acceleration

Detects short-term price momentum in BNB using pairs of lookback windows on 1-second candles from OKX.

  • Try lookback pairs: (7s, 10s) then (5s, 10s) then (5s, 7s)
  • Both returns must agree in direction (both positive or both negative)
  • At least one return must exceed the threshold: |return| ≥ 0.02%
  • First pair to pass fires the signal — Bull or Bear
Tier 2 — Fallback

BTC Confirmation

If Tier 1 doesn't fire, checks whether BNB and BTC are moving together as a confirmation signal.

  • BNB must have any nonzero 7-second return
  • BTC 30-second return must be in the same direction
  • BTC return must be significant: |return| ≥ 0.03%
  • Both must agree (BNB up + BTC up = Bull)

Runtime Loop — One Round
Epoch Alignment
Query the BSC smart contract for the current epoch number and round timing (when it locks, when it closes).
Sleep Until Cutoff
Calculate the decision deadline: 4 seconds before lock. Sleep until then (usually ~4.5 minutes).
~269 seconds of idle waiting
Prefetch Market Data
Immediately on wake: kick off two parallel background threads to fetch BNB and BTC 1-second candles from OKX. This runs while the next RPC calls happen.
Epoch Recheck + Pool Fetch
Quick RPC call to confirm the epoch hasn't shifted. Fetch current bull/bear pool amounts to inform sizing.
Compute Signal
Run the momentum gate: Tier 1 acceleration check, then Tier 2 BTC confirmation if needed. Result: Bull, Bear, or Skip.
Timing Guard + Execute
Final safety check: if less than 3 seconds remain before lock, abort (BSC blocks take ~3s). Otherwise, sign and broadcast the bet transaction.
Total decision window: ~1 second
Wait + Settle
Sleep until round closes (~5 more minutes). Check outcome via oracle price. If won, claim winnings on-chain. Loop to next round.
Operating Modes
--live

Live Mode

Real money. Reads wallet balance, signs transactions with a private key, broadcasts bets to BSC. Monitors outcomes and claims winnings automatically.

--dry

Dry Mode

Paper trading. Same signal pipeline and timing as live, but bets are simulated — no transactions hit the chain. Tracks a virtual bankroll to measure strategy performance risk-free.

--backtest

Backtest Mode

Offline historical replay. Loads thousands of past rounds and pre-cached 1s klines, replays the signal pipeline deterministically, and reports win rate and PnL.

--sync

Sync Mode

Data preparation. Fetches closed rounds from The Graph and 1s klines from OKX, caching them locally so backtest mode can run without live API calls.
